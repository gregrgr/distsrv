iOS 16+ 哪种证书才能签 mobileconfig？

iOS 严格要求签名证书的 Extended Key Usage 包含 Email Protection (S/MIME)。其它常见类型会被拒：

证书类型EKU能签 mobileconfig？
S/MIME (Email Protection)Email Protection✓ 是
Apple Distribution / DevelopmentCode Signing (critical)✗ 否
Let's Encrypt / 普通 TLS 证书Server Auth + Client Auth✗ 否
无 EKU 限制的证书(无)✓ 可能

S/MIME 证书来源（2024 年后基本都需要付费）

CA/Browser Forum 2024 S/MIME baseline 生效后，多数 CA 停了完全免费的个人 S/MIME（Actalis、Sectigo、Comodo 都已下线）。下面是目前还可用的来源：

  • HARICA（希腊学术 CA，公开可信）— 可能仍提供个人免费 email cert
    • https://www.harica.gr/en/email-certificate/
    • 需要注册账号 + email 验证
  • SSL.com（付费，约 $20 / 年）— 标准 personal S/MIME
    • https://www.ssl.com/certificates/email-smime/
  • DigiCert（付费，约 $30+ / 年）— 企业品牌可信度高
    • https://www.digicert.com/secure-email-smime-certificates

收到 .p12 + 密码后回到本页面上传。注意：CA 通常发 .p12 时邮件附密码，务必输入到下方"导出密码"字段。

不想买证书？用手动 UDID 提交（零成本，所有 iOS 都支持）

每个应用下载页 /d/<short_id> 都有"提交我的 UDID"按钮，配合 iOS 17+ 的剪贴板自动粘贴，用户操作只有 3 步：

  1. iPhone 设置 → 通用 → 关于本机 → 长按"序列号" → "复制 UDID"
  2. 切回 Safari，授权"允许此网站粘贴" → UDID 自动填入
  3. 点"提交 UDID" → 成功后跳回下载页，绿色横幅显示 UDID

这条路不依赖任何外部 CA，iOS 16 / iOS 26 都能用。如果你的内测设备不多，手动方式完全够用。

Mac 上把 S/MIME 证书导出成 .p12

Actalis 邮件附件本身就是 .p12，可以直接上传。如果是从其它 CA 拿到 PEM/cer + key 文件，或者你想从 macOS 钥匙串里转格式，按以下步骤：

  1. Mac 打开 钥匙串访问（Keychain Access）
  2. 双击 .cer 或 .pem 文件让证书安装到钥匙串
  3. 左栏选 登录 → "我的证书"
  4. 找到刚装的 S/MIME 证书，同时选中证书 + 私钥（按 Cmd 多选）
  5. 右键 → "导出 2 个项目" → 文件格式选 Personal Information Exchange (.p12)
  6. 设个密码，保存为 profile-signing.p12
  7. 把这个 .p12 上传到本页面
