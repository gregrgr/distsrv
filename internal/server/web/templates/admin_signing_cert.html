iOS 16+ 哪种证书才能签 mobileconfig？

iOS 严格要求签名证书的 Extended Key Usage 包含 Email Protection (S/MIME)。其它常见类型会被拒：

证书类型EKU能签 mobileconfig？
S/MIME (Email Protection)Email Protection✓ 是
Apple Distribution / DevelopmentCode Signing (critical)✗ 否
Let's Encrypt / 普通 TLS 证书Server Auth + Client Auth✗ 否
无 EKU 限制的证书(无)✓ 可能

免费 S/MIME 证书来源

  • Actalis（推荐，意大利 CA，全球受信）
    • 申请页面 — 提交邮箱 → 收激活码 → 下载 .p12 + 拿密码
    • 免费 1 年期；到期前再申请一份替换
    • 过程不到 5 分钟，无需身份验证
  • GlobalSign / Sectigo / DigiCert（付费，约 $15-30 / 年）— 用于正式生产

收到 .p12 后回到本页面上传，注意 Actalis 用邮件发的密码 → 输到下方的"导出密码"字段。

Mac 上把 S/MIME 证书导出成 .p12

Actalis 邮件附件本身就是 .p12，可以直接上传。如果是从其它 CA 拿到 PEM/cer + key 文件，或者你想从 macOS 钥匙串里转格式，按以下步骤：

  1. Mac 打开 钥匙串访问（Keychain Access）
  2. 双击 .cer 或 .pem 文件让证书安装到钥匙串
  3. 左栏选 登录 → "我的证书"
  4. 找到刚装的 S/MIME 证书，同时选中证书 + 私钥（按 Cmd 多选）
  5. 右键 → "导出 2 个项目" → 文件格式选 Personal Information Exchange (.p12)
  6. 设个密码，保存为 profile-signing.p12
  7. 把这个 .p12 上传到本页面
